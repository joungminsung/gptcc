# gptcc v2.4 — Modes & Settings Design

Status: Draft
Author: joungminsung (with Claude)
Date: 2026-04-20
Target release: gptcc v2.4.0 (public, npm)

---

## 1. Background & product goal

v2.3.1까지 gptcc는 단일 slot 구조. Claude Code의 `/model` picker에
GPT entry를 **추가**해 사용할 수 있게 하는 얇은 브릿지였음.
메모리에 박혀있는 제품 원칙: **"Claude Code main 경험은 건드리지 않고,
GPT를 pickable 옵션으로 추가만 한다."** v2.4도 이 원칙을 유지한다.

v2.4 목표:

1. 사용자가 `/model`에서 **`Auto`** 를 고르면 Opus가 main driver로 돌면서
   Sonnet / Haiku / GPT-5.4 에 task 유형별로 위임하도록 유도한다
   (orchestration-by-preset).
2. `/model`의 GPT 계열 entry를 단일화해서 Fastmode 토글로 `gpt-5.4 ↔
   gpt-5.4-fast` 가 바뀌도록 한다.
3. GPT 호출 경로(서브에이전트 프롬프트·변환 레이어·위임 프롬프트 규칙)에
   Anthropic식 어휘를 빼고 GPT-native 구조로 재작성한다.
4. "코드 수정마다 자동 리뷰(Auto review)"와 "위임 결과 재검증
   (SuperWork)" 을 독립 토글로 제공한다.
5. 모든 설정을 `~/.gptcc/config.json` 에 저장하고, 세 가지 방식으로 변경
   가능하게 한다: plugin slash command / `gptcc setting` CLI / 파일 직접
   편집.

Non-goals:

- Claude Code 본체의 `/model` entries (Default/Sonnet/Opus/Haiku) 를
  재정의하지 않는다. gptcc는 **추가**만 한다.
- Proxy-level 자동 라우팅(요청 분석해서 모델 자동 선택)은 하지 않는다.
  라우팅은 언제나 Claude 본체(또는 사용자)가 한다.
- OpenAI API key 기반 흐름은 기존과 동일 (ChatGPT OAuth만 공식 지원).

## 2. Architecture overview

```
Claude Code
  ├─ /model picker
  │    Default / Sonnet / Opus / Haiku    ← 기존, 건드리지 않음
  │    Auto / GPT-5.4                     ← gptcc v2.4 가 추가하는 2개
  │
  ├─ Plugin slash commands
  │    /gptcc-fastmode     on|off
  │    /gptcc-auto-review  on|off
  │    /gptcc-superwork    on|off
  │    /gptcc-setting              (현재 상태 조회)
  │
  ├─ Plugin hooks
  │    SessionStart  → 프록시 기동 (기존)
  │    PostToolUse   → Edit/Write 변경 파일 누적 (Auto review용)
  │    Stop          → Auto review · SuperWork 결과 검토 트리거
  │
  └─[HTTP 127.0.0.1:52532]──► gptcc proxy
                                 ├─ reads ~/.gptcc/config.json (mtime-cached)
                                 ├─ model routing:
                                 │    claude-auto-opus   → Opus + sys prompt 주입
                                 │    gpt-5.4 (Fastmode) → Codex (full or fast)
                                 │    claude-*           → Anthropic (unchanged)
                                 └─ 응답 변환 · stream

CLI: gptcc setting <key> <value> | list | reset
```

세 가지 표면만 새로 생긴다:

- `~/.gptcc/config.json` (상태 저장)
- `claude-auto-opus` 라는 sentinel model id (프록시가 의미 부여)
- plugin 안에 slash commands + 두 개의 hook 엔트리

## 3. /model picker 변경 사항

Claude Code의 `ANTHROPIC_CUSTOM_MODEL_OPTION` 은 picker에 **이름을
추가**하는 용도로만 쓴다. v2.4 가 추가하는 entries:

| Display | 내부 model id | 의미 |
|---|---|---|
| `Auto` | `claude-auto-opus` | Opus main + delegation skill 활성 |
| `GPT-5.4` | `gpt-5.4` (Fastmode off) / `gpt-5.4-fast` (on) | 단일 entry, Fastmode 토글에 따라 내부 라우팅 |

`v2.3.1` 에 있던 `gpt-5.4-fast` 별도 entry 는 v2.4에서 **제거**한다
(Fastmode 토글이 그 기능을 흡수). §11 호환성 참조.

## 4. Auto mode 메커니즘

Auto 는 실제 모델이 아니라 프록시 레벨의 **alias**. 동작 순서:

1. 사용자가 `/model` → `Auto` 선택.
2. Claude Code 는 `model: "claude-auto-opus"` 로 Anthropic API 호출
   (실제로는 `ANTHROPIC_BASE_URL` 덕분에 우리 프록시가 수신).
3. 프록시가 model 이름을 인지:
   - 실제 호출 모델을 `claude-opus-4-7` 로 치환.
   - 요청의 `system` 배열 앞쪽에 **delegation instruction block** 을
     한 덩어리 삽입 (§5).
   - SuperWork 가 on 이면 해당 block 마지막에 "GPT 위임 결과를
     Opus 가 재검토" 문단을 덧붙인다 (§9).
4. Opus 는 평소처럼 응답하되, 주입된 system prompt 덕분에 특정 task 유형
   에서 `Agent(subagent_type: ...)` 호출을 쉽게 선택한다.
5. sub-agent (gpt-reviewer / gpt-bug / gpt-arch / sonnet / haiku) 결과가
   돌아오면 Opus 가 통합해 최종 응답.

주입은 **transparent** 하다. Claude Code 나 사용자에게는 여전히 Opus 가
대화하는 것으로 보이고, /model 을 Default 로 바꾸면 주입 자체가 사라진다.

## 5. Delegation skill (Auto 프리셋 실체)

### 5.1 위치

`plugin/skills/gptcc-auto-delegate/SKILL.md`.

기존 `plugin/skills/orchestration/SKILL.md` 는 v2.4에서 다음 중 하나로 처리:

- 본 skill 로 **대체** (파일을 옮기고 내용 갱신) — 추천.
- 별도 유지 (Mode 2 / 수동 위임용). 이 경우 이름을 `gptcc-manual-delegate`
  로 구분.

### 5.2 트리거

`gptcc-auto-delegate` skill 의 본문은 파일 시스템에는 존재하되, Claude
Code 가 자동 로드하지는 **않는다**. 대신 프록시가 Auto 모드 요청
(`claude-auto-opus`) 을 감지할 때 해당 파일을 읽어 `system` 배열에
한 block 으로 주입한다. 즉 skill 본문 자체가 "Auto 모드 활성화"의 실체.

### 5.3 내용 — task → model mapping

공개 릴리스용으로 일반화했다 (사용자 개인 CLAUDE.md 규칙 그대로는
사용하지 않음):

| Task 유형 | 권장 위임 대상 | 왜 |
|---|---|---|
| 독립 모듈·파일 생성 (context 최소) | `gpt-5.4` (subagent) | 대화 컨텍스트 의존 낮음, anchoring 없는 독립 판단 |
| Diff 기반 코드 리뷰 | `gpt-5.4` | 다른 훈련·다른 시각 → overlap 가치 |
| 버그 근본 원인 분석 | `gpt-5.4` | 논리적 역추적, 독립 가설 |
| 아키텍처 second opinion | `gpt-5.4` | 시각 분산 |
| 중량 독립 생성 (여러 파일 묶음) | `sonnet` | Opus 대비 빠름, 복잡도 처리 가능 |
| 대량 반복 (유사 패턴 N개) | `haiku` | 비용 효율 |
| 기존 코드 이해·수정 | **Opus 본인** | 대화 컨텍스트·파일 상태 맥락 필요 |
| UI · 프론트엔드 · Figma MCP | **Opus 본인** | 시각 판단, 도구 연동 |
| 다중 턴 연속 대화 | **Opus 본인** | 컨텍스트 단절 회피 |
| 단순 Q&A · 설명 | **Opus 본인** | 위임 오버헤드 > 가치 |

Skill 의 어휘는 "참고하라"지 "강제하라"가 아니다. Opus 의 판단 여지를
남긴다. 잘못된 위임은 정답률보다 flow breakage 를 낳는다는 경험.

### 5.4 양식 규칙 (GPT subagent 를 부를 때)

Opus 가 Agent tool 로 GPT subagent 를 부를 때 따라야 하는 프롬프트
구조를 skill 안에서 명시한다 (§6의 3축 중 c 에 해당):

```
## Task     — GPT의 역할 ("You are performing an independent review")
## Goal     — 달성할 것
## Authoritative inputs
## Non-goals
## Output format
## Constraints  — 자료보다 먼저
<자료 (diff / symptoms / plan)>
```

금기:

- "be concise" + 깊은 분석을 동시에 요구
- "think step by step / carefully / deeply" 스택
- Claude 시스템 프롬프트·대화 transcript 원본 전달
- Claude 본인의 사전 가설 삽입 ("line 42가 문제 같다")

## 6. GPT Native 3축

### 6.1 Subagent prompts 재작성

대상 파일:

- `plugin/agents/gpt-reviewer.md`
- `plugin/agents/gpt-bug.md`
- `plugin/agents/gpt-arch.md`

각 파일을 §5.4 의 6-step 구조로 재작성한다. 공통 변경:

- Frontmatter 유지 (`model`, `tools`, `effort`).
- 본문의 어조를 Claude-style 에서 직접 명령형 ("You are performing …")
  으로 전환.
- `## Output format` 에 JSON / 스키마 / severity rubric 을 명시.
- `tools` 범위를 축소. 리뷰어는 `Read`·`Grep`·`Glob` 만, `Edit` 제외.
- "step by step", "be concise" 류 스택 제거.

### 6.2 Proxy 변환 레이어 개선 (`lib/proxy.mjs`)

Anthropic Messages API → OpenAI Responses API 변환 중:

- 요청 `system` 배열의 각 block 에 대해:
  - Claude-only tone guidance 스트립: 정규식 목록으로 `be concise`,
    `step by step`, `think carefully`, `think deeply` 등 지움.
  - markdown header 구조는 유지 (GPT 인식률이 더 좋다).
- reasoning effort 자동 결정 (요청 metadata 없으면):
  - 메시지·요청에 "review", "bug", "diagnose", "root cause" → `high`.
  - "generate", "write", "create" → `medium`.
  - 질의·설명 → `low`.
- 구현은 pure helper 로 분리 (`lib/gpt-native.mjs`) 해 테스트 가능하게.

### 6.3 Delegation prompt 구조 강제

§5.4 의 구조를 skill 안에 명시해 Opus 가 자연스럽게 따르게 한다.
Proxy 단에서 Agent tool 호출 payload 를 검사해 형식을 강제하지는
않는다 — 경고 로그만 해도 UX 에 노이즈가 되므로 skill 레벨의 유도만
사용한다.

## 7. Fastmode

토글. `~/.gptcc/config.json` 의 `fastmode: boolean` 필드.

동작:

- `false` (기본): /model `GPT-5.4` 선택 시 proxy 가 `gpt-5.4` 로 요청.
- `true`: 동일 entry 가 `gpt-5.4-fast` 로 라우팅.
- Auto 모드에서 GPT 서브에이전트(gpt-reviewer 등) 호출에도 동일 적용 —
  subagent frontmatter 의 `model: gpt-5.4-fast` 을 우리가 build 시
  `gpt-5.4` placeholder 로 바꾸고 proxy 가 config 에 따라 최종 결정.

구현:

- `lib/routing.mjs` 에 `resolveGptModel(requestedModel, config)` 순수
  함수 추가.
- `lib/proxy.mjs` 가 config read 후 `resolveGptModel` 로 실제 id 결정.
- Subagent frontmatter 는 `model: gpt-5.4-auto` 같은 sentinel 로 통일해
  proxy 가 resolver 를 통과시키게 한다.

## 8. Auto review

### 8.1 정의

Edit / Write 가 일어난 turn 이 끝나면 Stop hook 이 `gpt-reviewer`
subagent 를 호출해 변경 파일을 독립 리뷰. 결과는 Claude Code chat
메시지로 출력된다 (별도 UI 없이).

### 8.2 구현

- `plugin/hooks/hooks.json` 에 `PostToolUse`, `Stop` 추가:
  - `PostToolUse` on `Edit`·`Write`·`MultiEdit`: 변경 파일 경로를
    `~/.gptcc/state/pending-review.json` 에 append (JSONL).
  - `Stop`: config 의 `autoReview` 가 true 이고 pending 목록이 비어있지
    않으면 `gpt-reviewer` subagent 를 Agent tool 로 호출. 호출 후
    pending 파일 비우기.
- Hook 본체는 Node 스크립트 `plugin/hooks/auto-review.mjs`.
  - Stop hook 은 `exit 0` 하면서 stdout 으로 "additionalContext" 형태의
    리뷰 명령을 출력 (Claude Code 가 자동 턴 이어감).
- 리뷰 중 사용자가 새 요청 보내면 Stop hook 은 이미 종료됨 → 자연스럽게
  리뷰 결과가 한 turn 먼저, 새 요청이 다음 turn.

### 8.3 설정

- `autoReview`: boolean. 기본 false.
- `autoReviewMinLines`: integer. 기본 0. 누적 변경 라인이 이 값 미만이면
  리뷰 skip. (대량 변경만 리뷰하고 싶을 때 사용.)

## 9. SuperWork

### 9.1 정의

Auto 모드에서 **Opus 가 GPT subagent 에 위임한 결과를 받은 직후**, Opus
가 그 결과를 재검토하는 turn 을 한 번 더 유도한다. Auto review 와 독립.

### 9.2 구현

- Auto 모드 delegation skill 의 system prompt 끝에 조건부 block 을
  proxy 가 주입. `superWork` config 가 true 일 때만.
- block 내용 요약: "GPT subagent 결과를 받으면 즉시 자신의 판단과
  비교하여 동의·반대·중간 여부를 structured 하게 정리한 뒤 최종 응답을
  내라."
- Non-Auto 모드 (/model 이 Default/Sonnet/Opus/Haiku/GPT-5.4) 에서는
  no-op. skill block 이 주입되지 않기 때문에.

### 9.3 설정

- `superWork`: boolean. 기본 false.
- 추가 reasoning effort 조정 없음 (§6 의 자동 결정이 이미 high 를
  내는 task 유형에 주로 쓰이므로).

## 10. 토글 UX · config schema

### 10.1 config 파일

경로: `~/.gptcc/config.json`.

스키마:

```json
{
  "fastmode":            false,
  "autoReview":          false,
  "autoReviewMinLines":  0,
  "superWork":           false
}
```

- `gptcc setup` 은 파일이 없으면 default 값으로 생성.
- proxy 는 매 request 에서 `fs.stat` 로 mtime 확인, 변경 시 재로드
  (cheap; Node 캐시). 재시작 불필요.
- 알 수 없는 key 는 warn log + 무시.
- 파싱 실패 시 warn log + defaults.

### 10.2 CLI — `gptcc setting`

```
gptcc setting list
gptcc setting <key> <value>
gptcc setting reset
```

Keys (kebab-case on CLI, stored as camelCase in JSON):

- `fastmode`
- `auto-review`
- `auto-review-min-lines`
- `super-work`

Values accept `on|off|true|false` (booleans) or positive integer
(line count).

Output: pretty-print JSON after 각 write. Exit code 0 on success,
2 on validation error.

### 10.3 Slash commands

`plugin/commands/` 에 5 개 추가:

- `gptcc-fastmode.md`
- `gptcc-auto-review.md`
- `gptcc-superwork.md`
- `gptcc-setting.md`
- (선택) `gptcc-help.md`

구현은 각 `.md` 내부에서 `!gptcc setting ...` bash 호출. 즉 slash
command 는 CLI 를 얇게 감싼다.

### 10.4 우선순위

동일 키를 여러 곳에서 바꾸면 **가장 최근 파일 쓰기**가 항상 승. 즉 3
가지 방식은 모두 `~/.gptcc/config.json` 을 대상으로 read/modify/write
하므로 race 는 파일 system 수준에서만 일어남. POSIX mv 기반 atomic
write 사용.

## 11. v2.3 → v2.4 호환성 · 마이그레이션

### 11.1 변경 사항

- /model 에서 `gpt-5.4-fast` entry 제거 (Fastmode 로 대체).
- /model 에 `claude-auto-opus` ("Auto") entry 신설.
- `plugin/skills/orchestration` → `plugin/skills/gptcc-auto-delegate` 로
  이름/내용 변경 (또는 별도 유지, §5.1).
- `plugin/agents/*.md` 본문 전면 재작성.
- `~/.gptcc/config.json` 신설 (없으면 생성, 있으면 존중).

### 11.2 사용자 영향

- `/model` 에서 `gpt-5.4-fast` 를 고르던 사용자: v2.4 설치 후에는 entry
  가 사라지므로 `gpt-5.4` 를 고르고 Fastmode 토글 on 으로 전환. `gptcc
  setup` 시 감지해 안내 메시지 1 회 출력.
- Mode 2 (수동 Agent 위임) 사용자: `gpt-reviewer/bug/arch` 이름은 그대로.
  프롬프트 내용만 바뀌므로 사용 방법 무변경.
- 롤백: `npm install -g gptcc@2.3.1` 로 이전 버전 사용 가능. config 파일
  존재해도 v2.3 은 무시.

### 11.3 `gptcc setup` 마이그레이션 로직

1. 기존 Claude Code settings 의 `ANTHROPIC_CUSTOM_MODEL_OPTION` 에서
   `gpt-5.4-fast` entry 발견 시: 제거 후 안내 메시지.
2. `claude-auto-opus` entry 가 없으면 추가.
3. `~/.gptcc/config.json` 없으면 default 로 생성.

## 13. 에러 처리 · fallback

- `~/.gptcc/config.json` 파싱 실패: warn + defaults. 서비스 중단 없음.
- 파일 없음: default 로 생성.
- Auto 모드 subagent 호출 실패 (proxy 에러·OAuth 만료 등): Opus 에게
  에러를 노출 (tool error content). Opus 가 "GPT 위임 실패. 직접 진행
  한다"로 자연스럽게 이어감.
- SuperWork 재검토 중 context overflow: Opus 가 재검토를 포기하고
  원래 subagent 결과를 그대로 내놓도록 skill 문구에 명시.
- Auto review 중 proxy 다운: Stop hook 이 fetch 실패 시 silent skip.
  리뷰가 누락되더라도 대화 진행은 막지 않는다.

## 14. 보안·권한 영향

- 새 파일 쓰기 위치는 `~/.gptcc/` 하위로 제한. `gptcc setup` 이 이미
  쓰는 경로.
- slash command / CLI 는 추가 권한 요청 없음 (사용자 디렉토리).
- proxy 는 기존 `127.0.0.1:52532` bind. 변경 없음.
